# RisuAI LLM 스킬 라이브러리 통합 번역본

이 문서는 AI 어시스턴트 및 사용자를 위해 최적화된 `@skills` 디렉토리 내의 영어 가이드 문서들을 자연스러운 한국어로 번역 및 요약한 통합 문서입니다.

---

## 1. 캐릭터 제작 가이드 (Authoring Characters)

LLM 롤플레잉을 위한 캐릭터 설명을 작성하고 리팩토링하는 가이드입니다. 키워드부터 완전한 설정까지 모든 입력을 처리하여 행동의 깊이, 대화 레지스터(말투/어조) 시스템, 대조적인 특성, 반응 패턴을 갖춘 실전용 캐릭터를 만듭니다.

### 핵심 원칙

1. **설명은 곧 연기 지시서입니다:** 모든 문장은 "이것이 LLM의 다음 출력(행동/대사)을 어떻게 변화시키는가?"에 답해야 합니다.
2. **단순한 꼬리표 대신 행동으로 묘사하세요:** "겉은 차갑지만 속은 따뜻하다"라고 쓰지 마세요. 차갑게 느껴지는 구체적인 행동과, 따뜻함이 무심코 묻어나는 순간을 묘사하세요.
3. **대화(말투)에 투자하세요:** 외모 묘사는 한 번 쓰이지만, 대화는 매 턴마다 등장합니다. 예시 대사와 대화 패턴은 LLM의 출력을 형성하는 가장 강력한 도구입니다.
4. **전략적인 공백을 남겨두세요:** 캐릭터가 누구인지, 어떻게 행동하는지는 정의하되, 앞으로 무슨 일이 일어날지, 누구와 사랑에 빠질지 등 미래의 감정이나 사건은 규정하지 마세요.
5. **내적 갈등이 깊이를 만듭니다:** 캐릭터의 과거에 기반하여, 서로 다른 방향으로 당기는 모순된 특성을 부여하세요.

### 제작 파이프라인

1. **핵심 동기(Core Drive) 찾기:** "왜 그것이 그들에게 중요한가?"를 계속 질문하여 표면적인 욕구 아래의 근본적인 결핍이나 필요(예: "나를 상처입힐 수 있는 모든 불확실성을 통제하고 싶다")를 찾아냅니다.
2. **대조적 쌍(Contrast Pairs) 설계:** 겉으로 보여지는 모습(표면)과 그 이면의 실제 모습(현실)을 정의하고, 이 둘이 공존하는 이유를 과거사와 연결합니다. (예: 완벽을 연기하는 얼음공주 vs 사실은 작은 농담에도 크게 웃고 누군가 진짜 모습을 알아봐주길 바라는 내면)
3. **대화 시스템 구축:** 2~6개의 대화 레지스터(상황/감정에 따른 어조 변화)를 정의하고, 각 레지스터마다 3~5개의 예시 대사와 행동 지문을 포함합니다.
4. **내면의 목소리 정의:** 캐릭터의 속마음(독백)이 실제 밖으로 내뱉는 말과 얼마나 다른지(Speech gap), 스스로에게 얼마나 솔직한지를 정의합니다.
5. **반응 패턴 및 심리적 깊이 매핑:** 진심 어린 친절, 직접적인 비판, 스킨십 등 핵심 감정 트리거에 대한 본능적인 반응 패턴을 정의합니다.

## 2. 대화 시스템 레퍼런스 (Speech System)

캐릭터 설명에서 가장 많은 투자가 필요한 부분입니다. 상황적 이름(예: "화났을 때")이 아닌 심리적 이름(예: "방패 - 상처를 건드리면 나오는 방어기제")으로 레지스터를 명명하세요.

### 각 레지스터 필수 요소

1. **발동 조건:** 이 어조가 언제 튀어나오는지 묘사합니다.
2. **예시 대사 (3~5개):** 괄호 안에 행동 지문(Stage directions)을 포함한 실제 대사 예시를 작성합니다.
   - _예: "(팔찌를 만지작거리며 시선을 피한다) ...엄마는 원래 향신료는 눈물이 찔끔 날 정도로 넣어야 한다고 하셨어."_
3. **비언어적 특징:** 해당 레지스터일 때의 자세, 시선 처리, 손동작 등을 묘사합니다.
4. **언어적 특징:** 문장 길이, 어휘 수준, 말버릇, 유머나 비꼬는 말투의 사용 여부를 정의합니다.

### 캐릭터 DNA (일관성 앵커)

모든 레지스터에 걸쳐 **항상 존재하는** 2~4가지의 언어적/행동적 버릇(예: 항상 손으로 무언가를 만지작거림, 대답 대신 질문으로 응수함)을 정의하여 캐릭터가 감정 변화 속에서도 일관성을 유지하게 합니다.

## 3. 캐릭터 검증 (Validation)

캐릭터 완성을 위한 체크리스트입니다.

- **행동 예측 가능성:** 설명만 보고도 임의의 3가지 상황에서 캐릭터의 반응을 예측할 수 있습니까?
- **대화 투자:** 예시 대사가 충분히 포함되어 있습니까?
- **대조의 존재:** 겉모습과 내면의 대조(Surface vs Subversion)가 뚜렷합니까?
- **전략적 공백:** 미래의 감정이나 전개를 강제하는 문장("결국 사랑에 빠진다" 등)이 없습니까?
- **현재 상태:** RP가 시작되는 시점의 감정 상태나 직면한 긴장감이 있습니까?

## 4. 로어북 기반 봇 제작 (Authoring Lorebook Bots)

세부적인 세계관/캐릭터 정보가 로어북에 저장되는 봇의 메인 설명(Description)을 작성하는 가이드입니다.

**핵심 원칙:** 메인 설명은 위키백과가 아니라 **분위기를 형성하는 렌즈(Interpretive lens)**입니다.

- 역사 수업, 정치 체제, 지리 정보는 로어북으로 넘기세요.
- 메인 설명에는 **세계의 분위기, 핵심 규칙, 캐릭터의 본질, 현재의 감정 상태, 그리고 말투의 DNA**만 남깁니다.
- 설명란의 문체는 LLM이 RP에서 사용할 문체의 기준이 됩니다. 분위기 있는 문장으로 작성하세요.

### 로어북 아키텍처 (Lorebook Architecture)

- **독립성:** 각 로어북 항목은 단독으로 읽어도 이해될 수 있어야 합니다.
- **분위기 일치:** 메인 설명이 소설 같다면, 로어북 항목도 기계적인 설명이 아닌 소설적인 분위기로 작성해야 합니다.
- **전기(Biography) 대신 행동(Behavior):** "그는 45세이고 흉터가 있다"보다 "그는 언제 끝날지 모를 유배지에서 매일 무기를 손질하는 융통성 없는 장교다"처럼 행동에 초점을 맞추세요.
- **트리거 깊이:** 가장 깊은 비밀은 캐릭터의 이름이 아니라, 신뢰가 쌓였을 때만 등장할 법한 특정 대화 주제(키워드)와 연결하세요.

## 5. 에셋 프롬프트 작성 (Writing Asset Prompts)

Anima 모델에 최적화된 캐릭터 스탠딩 일러스트 프롬프트를 작성하는 6단계 파이프라인입니다.

1. **기본 구성:** `1girl`, `cowboy shot`(허벅지 위) 등 구도 설정.
2. **표정 & 포즈:** 캐릭터의 성격이 드러나는 포즈(`arms crossed`, `cheerful smile`).
3. **의상:** 겉옷부터 속옷, 재질까지 상세히 묘사.
4. **소품 & 효과:** 들고 있는 무기나 홀로그램 등 특수 효과.
5. **조명 & 미니멀 배경:** `(minimalist solid pale green background:1.2)`, `simple background` 등 채팅 UI에 어울리는 단색/심플 배경 강제.
6. **자연어 요약 (가장 중요):** 프롬프트 맨 마지막에 캐릭터를 묘사하는 완벽한 영어 문장 한 줄을 추가합니다. (예: `A cheerful hacker girl in oversized techwear, holding a glowing holographic tablet.`)

## 6. RisuAI 문법 및 스크립팅 요약

### CBS 문법 (Custom Bracket Syntax)

`{{tag::arg1::arg2}}` 형태의 런타임 템플릿 문법입니다.

- 변수 제어: `{{getvar::hp}}`, `{{setvar::hp::100}}`
- 조건문: `{{#when::{{getvar::hp}}::<::20}} ... {{:else}} ... {{/when}}`
- 주사위/수식: `{{roll::2d6}}`, `{{calc::10+5}}`
- 캐릭터 데이터: `{{char}}`, `{{user}}`, `{{description}}`
- **주의사항:** `setvar`와 같은 쓰기 함수는 `runVar` 컨텍스트(첫 메시지, 로어북 활성화 시 등)에서만 동작합니다.

### Lua 스크립트

RisuAI `.charx`, `.risum` 파일의 `lua` 필드에 작성하는 Lua 5.4 기반 트리거 스크립트입니다.

- **이벤트 훅:** `onInput(id)` (사용자 입력 시), `onStart(id)` (프롬프트 생성 전), `onOutput(id)` (AI 응답 후).
- **비동기 처리:** LLM 호출(`LLM()`, `simpleLLM()`), UI 알림(`alertInput()`) 등은 반드시 `:await()`를 붙여야 합니다.
- **상태 관리:** `getState(id, "key")`, `setState(id, "key", value)`를 사용하면 테이블(배열/객체)을 자동으로 JSON 직렬화하여 저장합니다.

### HTML 및 CSS 작성

- **CSS 위치:** 스타일은 반드시 `backgroundEmbedding` 필드에 `<style>` 태그로 선언하세요. 정규식 OUT 필드에 넣으면 매치될 때마다 스타일이 중복 주입됩니다.
- **클래스 접두사:** RisuAI는 렌더링 시 모든 CSS 클래스에 `x-risu-` 접두사를 붙입니다. 체이닝 선택자(`.status.x-risu-active`)를 쓸 때는 수동으로 접두사를 붙여야 합니다.
- **공백 줄 금지:** HTML 태그 사이에 빈 줄을 넣으면 마크다운 파서가 구조를 깨뜨리므로 태그를 연속해서 작성하세요.

### 정규식 스크립트 (Regex Scripts)

자바스크립트 정규식을 사용해 텍스트를 변환합니다.

- **타입:** `editInput` (사용자 입력 변환), `editOutput` (AI 응답 변환), `editDisplay` (렌더링만 변경, 데이터 보존), `editRequest` (프롬프트에만 몰래 주입).
- **특수 OUT 프리픽스:** `@@emo 감정이름` (감정 이미지 표시), `@@move_top` (매치된 텍스트를 맨 위로 이동).

### 트리거 스크립트 (Trigger Scripts)

GUI 트리거 또는 Lua를 통해 챗 이벤트 발생 시 자동화된 동작을 수행합니다.

- **파이프라인 순서:** CBS 파싱 → 트리거 스크립트 실행 → CBS 2차 파싱 → 정규식 스크립트 적용.
- 트리거 로직(Lua 등) 내부에서는 CBS `{{}}` 구문이 파싱되지 않으므로, 반드시 Lua 전용 API(`getState`, `getChatVar`)를 사용해야 합니다.
