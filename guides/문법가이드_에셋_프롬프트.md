# 문법가이드 정규식

정규식을 활용한 채팅 입력, 출력, 프롬프트, UI의 동적 제어

---

## 정규식 스크립트란?

**정규식(Regular Expression)**을 사용하여 채팅의 모든 흐름을 가로채고 수정할 수 있는 강력한 기능입니다. 단순한 오타 수정부터 복잡한 게임 시스템 구현까지 가능합니다.

---

## 스크립트 구성 필드

### 이름

스크립트의 관리 및 식별을 위한 제목입니다.

예: "이모티콘을 텍스트로 변환", "행동 묘사 강조"

### IN

매칭할 **JavaScript 정규식 패턴**입니다.

예: `:(\\)|\\(|D|P|O)`

- **CBS 사용:** flag에 `IN CBS Parsing`을 포함하면 CBS 문법을 사용할 수 있습니다.

### OUT

매칭된 내용이 변경될 **출력 값**입니다.

- HTML 태그를 자유롭게 사용할 수 있습니다.
- CBS 문법(`{{getvar::변수명}}` 등)을 지원합니다.

### Modification Type

스크립트가 적용되는 **시점**을 결정합니다. (자세한 내용은 아래 섹션 참조)

### flag

정규식 플래그 및 특수 기능을 설정합니다.

**[정규식 플래그]**
| 플래그 | 설명 |
|--------|------|
| g | 전역 매칭 (모든 항목) |
| i | 대소문자 무시 |
| m | 멀티라인 모드 |
| s | dotAll 모드 (.이 줄바꿈 포함) |
| u | 문자를 유니코드로 치환 |

**[특수 플래그]**
- `IN CBS Parsing`: IN 필드에서 CBS 사용 활성화
- `Move Top/Bottom`: 매칭 내용을 맨 위/아래로 이동
- `Repeat Back`: 원본 뒤에 반복
- `Order Flag`: 실행 우선순위 지정 (숫자가 높을수록 먼저 실행)

### Custom Flag

- `true`: flag 필드에 작성된 내용을 적용합니다.
- `false`: 내용을 무시하고 기본값(g, order 0)을 적용합니다.

## 타입(Type) 상세 가이드

### 입력문 수정

사용자가 입력한 텍스트를 **서버로 전송하기 전에** 수정합니다.

**활용:** 오타 자동 교정, 특정 명령어 단축키, 이모티콘 변환

### 출력문 수정

AI가 생성한 응답 텍스트를 **화면에 뿌리기 전(생성 후)**에 수정합니다.

**활용:** 행동 묘사(*...*) 스타일링, 특정 단어 필터링, 말풍선 효과 적용

### 리퀘스트 데이터 수정

LLM에 전송되는 **최종 프롬프트**를 수정합니다. (채팅 로그는 수정하지 않음)

**활용:** 시스템 프롬프트 주입, 특정 문구 프롬프트 제거

### 디스플레이 수정

**채팅 화면에 보여지는 내용만** 수정합니다. (실제 데이터는 변하지 않음)

**활용:** 상태창 렌더링, 시각적 연출, UI 요소 추가

### 번역문 수정

번역기를 거친 결과물을 **최종 반영 전에** 수정합니다.

**활용:** 고유명사 오역 수정, 번역체 말투 교정

## 고급 기능: CBS & HTML

CBS (Curly Braced Syntaxes)는 스크립트 내에서 변수 호출, 계산, 랜덤 함수 등을 사용할 수 있게 해주는 문법입니다.

- CBS 상세: [CBS_QUICK_REF.md](CBS_QUICK_REF.md) 참고
- HTML/CSS 상세: [문법가이드 HTML/CSS](문법가이드_HTML_CSS.md) 참고

### OUT 필드 작성 예시 (상태창 구현)

```html
<div class="status-box">
  <h3>{{getvar::char_name}}</h3>
  <p>HP: {{getvar::hp}} / {{getvar::max_hp}}</p>
  <p>주사위 결과: {{roll::1d20}}</p>
</div>
```
