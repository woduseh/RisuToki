# 문법가이드 트리거 스크립트

이벤트 기반 자동화 및 동적 제어 스크립트

---

## 트리거란?

채팅 전송, 수신, 버튼 클릭 등 **특정 이벤트가 발생했을 때 자동으로 실행**되는 스크립트입니다. 변수 조작, UI 변경, 프롬프트 제어 등 다양한 작업을 수행할 수 있습니다.

---

## 작동 순서 및 주의사항

```
CBS 파싱 (변수 치환)
트리거 스크립트 실행
CBS 파싱 (재확인)
정규식 스크립트 (내부 CBS 파싱 포함)
```

### CBS 사용 주의

트리거 스크립트 **내부 로직**에서는 CBS가 파싱되지 않습니다.

- `if {{getvar::hp}} > 10 then` → 작동 안 함
- 채팅창에 출력하거나 데이터를 수정하여 내보내는 **문자열**에는 CBS를 포함할 수 있음

## 트리거 이벤트 종류

### 1. 채팅 보낼 시 (Start)

사용자가 전송 버튼을 누르면 발동합니다. **프롬프트 생성 전에 실행**되므로 프롬프트에 영향을 줄 수 있습니다.

**V2 설정:** 활성화 조건 → 채팅 보낼 시

```lua
function onStart(id)
  -- 이벤트 내용 작성
end
```

### 2. 봇 출력 시 (Output)

AI의 답변 생성이 완료되면 발동합니다. 출력문을 수정하거나 턴 종료 이벤트를 처리할 때 사용합니다.

**V2 설정:** 활성화 조건 → 봇 출력 시

```lua
function onOutput(id)
  -- 이벤트 내용 작성
end

-- 혹은 감지 리스너 사용
listenEdit("editOutput", function(triggerId, data)
  -- data: 출력문 텍스트
end)
```

### 3. 유저 입력 시 (Input)

유저의 입력이 확정되어 채팅창에 올라갈 때 발동합니다. **프롬프트 생성 후에 실행**되므로 프롬프트에는 영향을 주지 않습니다.

**V2 설정:** 활성화 조건 → 유저 입력 시

```lua
function onInput(id)
  -- 이벤트 내용 작성
end

-- 혹은 감지 리스너 사용
listenEdit("editInput", function(triggerId, data)
  -- data: 입력문 텍스트
end)
```

### 4. 수동 트리거 (Manual/Button)

버튼 클릭, 다른 트리거에서의 호출 등 **명시적인 호출**로 발동합니다.

**실행 방법:**
- 다른 트리거에서 함수 호출
- CBS: `{{button::Display::TriggerName}}`
- HTML: `<div risu-trigger="TriggerName">버튼</div>`

```lua
-- 일반 수동 실행
local function onButton(triggerId)
  -- 버튼 클릭 시 처리
end

-- risu-btn="데이터" 속성으로 대체 가능
function onButtonClick(id, data)
  -- data: risu-btn 속성의 값
end
```

### 5. 디스플레이 & 리퀘스트 수정

데이터를 가로채서 화면 표시(UI)나 전송 데이터(Prompt)를 수정합니다.

#### 디스플레이 수정 (editDisplay)

채팅 데이터 변경 없이 **UI만 변경** (상태창, 텍스트 숨김 등)

```lua
listenEdit("editDisplay", function(triggerId, data)
  -- data: 디스플레이될 HTML 내용
  return data .. "<div>추가됨</div>"
end)
```

#### 리퀘스트 데이터 수정 (editRequest)

채팅 데이터 변경 없이 **프롬프트 삽입 내용만 수정**

```lua
listenEdit("editRequest", function(triggerId, data)
  -- data: 프롬프트에 들어갈 텍스트
  return "[시스템 메시지 주입] " .. data
end)
```
