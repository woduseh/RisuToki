# 정규식 최적화 가이드

**용도:** 토큰 절약 / 로딩 렉 감소

---

## 핵심 원리

```
{{#if {{greater_equal::{{chat_index}}::{{? {{lastmessageid}}-5}}}}}}
$&
{{/if}}
```

- **최근 5개 메시지만 렌더링**, 그 이전은 숨김
- `$&` = 매칭된 전체 내용 그대로 출력
- `-5` 숫자 조절 가능 (더 많이/적게)

---

## 1. 이미지 에셋 리퀘 제거

### IN 패턴 만들기

봇마다 다른 이미지 명령문 형식:
- `<img src="...">`
- `<img="...">`
- `<sex img="...">`

**IN 작성법:**
```
# 형식이 <img src="hello"> 면
<img src="(.*?)">

# 형식이 <img src=hello> 면
<img src=(.*?)>
```

### OUT
```
{{#if {{greater_equal::{{chat_index}}::{{? {{lastmessageid}}-5}}}}}}
$&
{{/if}}
```

---

## 2. 상태창 리퀘 제거 (가장 중요)

상태창은 **최대 10줄**까지 쌓임 → 토큰 폭탄

### IN 패턴 만들기

상태창 시작/끝 태그 확인:
```
# 예: <Char_Status>...</Char_Status>
<Char_Status>[\s\S]*?</Char_Status>

# 예: {STATUS|...}
\{STATUS\|[\s\S]*?\}
```

**`[\s\S]*?`** = 줄바꿈 포함 모든 내용

### OUT
```
{{#if {{greater_equal::{{chat_index}}::{{? {{lastmessageid}}-5}}}}}}
$&
{{/if}}
```

### 주의
장기 상태 저장용 상태창(민감도, 절정 횟수, 아이템, 날짜 등)은 **리퀘 제거하지 말 것**
→ 요약 시 컨텍스트에 포함되어야 함

---

## 3. 로딩 렉 줄이기 - 간단한 방법

기존 에셋 정규식 OUT에 if 구문 추가:

### 변경 전
```
<div class="status">...</div>
```

### 변경 후
```
{{#if {{greater_equal::{{chat_index}}::{{? {{lastmessageid}}-5}}}}}}
<div class="status">...</div>
{{/if}}
```

---

## 4. 로딩 렉 줄이기 - 더 나은 방법

**새 정규식을 만들어서 기존 정규식보다 위에 배치**

### 원리
정규식은 **위 → 아래** 순서로 적용됨

**비효율적 순서:**
1. 상태창 정규식 찾기
2. 5개 이전이면 숨김
3. 출력

**효율적 순서:**
1. 5개 이전 상태창 **먼저 찾아서 제거**
2. 그 다음 출력

### 적용 방법
1. 새 정규식 생성
2. IN: 기존 상태창 IN 패턴
3. OUT: if 구문 + `$&`
4. **기존 상태창 정규식보다 위에 배치**
