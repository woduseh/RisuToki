# 핵심 패턴 가이드

> RisuAI 봇을 만들 때 반복적으로 쓰이는 핵심 패턴들입니다.
> 각 문법 가이드(Lua, 정규식, CBS 등)를 읽기 전에 이 문서를 먼저 읽으면 전체 구조를 이해할 수 있습니다.

---

## 전체 구조: 모든 것의 흐름

RisuAI 봇은 5개 레이어가 순서대로 동작합니다:

```
[유저가 메시지 전송]
  ↓
① Lua onStart()        ← 프롬프트 생성 전 변수 처리
  ↓
② AI가 응답 생성       ← 글로벌노트 + 로어북이 프롬프트에 주입됨
  ↓
③ Lua onOutput()       ← AI 응답에서 태그 파싱, 변수 업데이트
  ↓
④ 정규식 (editDisplay) ← 화면에 보이는 내용을 HTML로 변환
  ↓
⑤ 유저 화면에 표시     ← CSS로 스타일링
```

**핵심**: Lua가 데이터를 처리하고, 정규식이 화면을 그립니다. 이 두 가지가 전부입니다.

## 패턴 1: 버튼 → 변수 → 화면 갱신 (가장 기본)

RisuAI 인터랙티브 봇의 **가장 핵심적인 패턴**입니다. 모든 UI 상호작용이 이 루프를 따릅니다.

```
[유저가 버튼 클릭]
  ↓
Lua: onButtonClick(triggerId, data)  ← 버튼 값 수신
  ↓
Lua: setChatVar(triggerId, "cv_변수", "값")  ← 상태 변경
  ↓
Lua: reloadDisplay(triggerId)  ← 화면 갱신 요청
  ↓
정규식(editDisplay): {{#if {{equal::{{getvar::cv_변수}}::값}}}} → HTML 렌더링
```

### 실전 예시: 시작 버튼

**1단계: HTML (퍼스트 메시지)**
```html
{{#if {{equal::{{getvar::cv_step}}::0}}}}
<div risu-btn="start-btn" style="padding:12px;cursor:pointer">시작하기</div>
{{/if}}
{{#if {{equal::{{getvar::cv_step}}::1}}}}
<div>게임이 시작되었습니다!</div>
{{/if}}
```

**2단계: Lua (버튼 핸들러)**
```lua
function onButtonClick(triggerId, data)
    if data == "start-btn" then
        setChatVar(triggerId, "cv_step", "1")
        reloadDisplay(triggerId)
        return
    end
end
```

**3단계: 기본변수**
```
cv_step: 0
```

이것만으로 "시작하기" 버튼을 누르면 화면이 바뀌는 인터랙티브 UI가 완성됩니다.

## 패턴 2: Step 기반 UI (다단계 화면 전환)

하나의 정규식에서 CBS 조건문으로 여러 화면을 전환하는 패턴입니다.

### 흐름
```
step "0" → 시작 화면
step "1" → 선택 화면
step "2" → 확인 화면
step "3" → 게임 시작
```

### 정규식 (editdisplay)
**IN:** `\{GAME_UI\}`
**OUT:**
```html
{{#if {{equal::{{getvar::cv_step}}::0}}}}
<div>시작 화면 HTML</div>
<div risu-btn="go-step1">다음</div>
{{/if}}
{{#if {{equal::{{getvar::cv_step}}::1}}}}
<div>선택 화면 HTML</div>
<div risu-btn="select-A">A 선택</div>
<div risu-btn="select-B">B 선택</div>
{{/if}}
{{#if {{equal::{{getvar::cv_step}}::2}}}}
<div>선택 완료: {{getvar::cv_selected}}</div>
<div risu-btn="confirm">확정</div>
<div risu-btn="go-step1">다시 선택</div>
{{/if}}
```

### Lua
```lua
function onButtonClick(triggerId, data)
    if data == "go-step1" then
        setChatVar(triggerId, "cv_step", "1")
        reloadDisplay(triggerId)
        return
    end
    if data == "select-A" then
        setChatVar(triggerId, "cv_selected", "A")
        setChatVar(triggerId, "cv_step", "2")
        reloadDisplay(triggerId)
        return
    end
    if data == "select-B" then
        setChatVar(triggerId, "cv_selected", "B")
        setChatVar(triggerId, "cv_step", "2")
        reloadDisplay(triggerId)
        return
    end
    if data == "confirm" then
        setChatVar(triggerId, "cv_step", "3")
        reloadDisplay(triggerId)
        return
    end
end
```

**핵심**: `cv_step` 하나로 전체 UI 흐름을 제어합니다.

## 패턴 3: AI 응답 태그 파싱

AI가 특정 형식의 태그를 출력하면 Lua가 파싱하여 게임 상태를 업데이트하는 패턴입니다.

### 흐름
```
[글로벌노트에서 AI에게 태그 출력 지시]
  ↓
AI 응답: "검을 휘둘렀다. {DAMAGE|30} 적이 비틀거린다. {STATUS|hp=70|mp=50}"
  ↓
Lua onOutput(): 태그 파싱 → setChatVar
  ↓
정규식 editDisplay: 태그를 화면에서 숨기거나 UI로 변환
정규식 editRequest: 태그를 프롬프트에서 제거 (토큰 절약)
```

### 글로벌노트 (AI 지시)
```
When dealing damage, output: {DAMAGE|숫자}
When updating status, output: {STATUS|hp=숫자|mp=숫자}
```

### Lua (태그 파싱)
```lua
function onOutput(triggerId)
    local msg = getCharacterLastMessage(triggerId)

    -- {DAMAGE|30} 파싱
    for dmg in msg:gmatch("{DAMAGE|(%d+)}") do
        local hp = tonumber(getChatVar(triggerId, "cv_hp") or "100")
        hp = hp - tonumber(dmg)
        setChatVar(triggerId, "cv_hp", tostring(hp))
    end

    -- {STATUS|hp=70|mp=50} 파싱
    for content in msg:gmatch("{STATUS|([^}]+)}") do
        for key, val in content:gmatch("(%w+)=(%w+)") do
            setChatVar(triggerId, "cv_" .. key, val)
        end
    end
end
```

### 정규식 — 태그 숨김 (editdisplay)
**IN:** `\{DAMAGE\|\d+\}` → **OUT:** (빈 문자열)
**IN:** `\{STATUS\|[^}]+\}` → **OUT:** (빈 문자열)

### 정규식 — 프롬프트에서 제거 (editrequest)
토큰 절약을 위해 오래된 태그를 프롬프트에서 제거:
**IN:** `\{STATUS\|[^}]+\}` → **OUT:**
```
{{#if {{greater_equal::{{chat_index}}::{{? {{lastmessageid}}-5}}}}}}$&{{/if}}
```
최근 5개 메시지만 유지, 이전 태그는 프롬프트에서 삭제됩니다.

## 패턴 4: 동적 로어북 (Lua로 활성화/생성)

로어북을 미리 만들어 두되 비활성화(`@@dont_activate`)하고, 특정 조건에서 Lua로 활성화하는 패턴입니다.

### 용도
- 캐릭터가 등장할 때만 해당 캐릭터의 설정을 프롬프트에 주입
- 이벤트 발생 시 관련 로어북 로드
- 유저 입력으로 캐릭터를 선택하면 해당 RP 지침 활성화

### 방법 1: 기존 로어북 검색 후 복사

```lua
-- DB에서 로어북 검색 (comment 기준)
local results = getLoreBooks(triggerId, "캐릭터명")
for _, book in ipairs(results) do
    if book.comment == "캐릭터명 (RP)" then
        -- @@dont_activate 제거 후 로컬 로어북으로 생성
        local content = book.content:gsub("@@dont_activate\n", "")
        upsertLocalLoreBook(triggerId, "active_character_rp", content, {
            key = "캐릭터명",
            alwaysActive = true
        })
        break
    end
end
```

### 방법 2: Lua에서 직접 생성

```lua
-- 유저 입력을 기반으로 로어북 생성
local name = getChatVar(triggerId, "cv_player_name") or ""
local content = "[Player Profile]\nName: " .. name .. "\nRole: Adventurer"

upsertLocalLoreBook(triggerId, "player_profile", content, {
    key = name,
    alwaysActive = true
})
```

### 주의사항
- `upsertLocalLoreBook`은 **1턴 지연**이 있음 — 생성 직후의 AI 응답에는 반영 안 될 수 있음
- 같은 name으로 다시 호출하면 기존 로어북을 **덮어씀** (upsert)

## 패턴 5: 비동기 유저 입력 (alertInput / alertSelect)

버튼 클릭이 아닌 텍스트 입력이나 선택지가 필요할 때 사용합니다.

### 주의: async 래퍼 필수

`alertInput`, `alertSelect`, `alertConfirm`은 비동기 함수이므로 반드시 `async` 래퍼 안에서 `:await()`와 함께 사용합니다.

### 텍스트 입력
```lua
-- risu-trigger로 호출되는 함수
function editPlayerName(triggerId)
    async(function(triggerId)
        local result = alertInput(triggerId, "이름을 입력하세요:"):await()
        if result and result ~= "" and result ~= "null" then
            setChatVar(triggerId, "cv_player_name", result)
            reloadDisplay(triggerId)
        end
    end)
end
```

**HTML에서 호출:**
```html
<div risu-trigger="editPlayerName">이름 수정</div>
```

> `risu-trigger="함수명"` — 해당 Lua 함수를 직접 호출
> `risu-btn="값"` — `onButtonClick(triggerId, "값")`을 호출

### 선택지
```lua
function selectClass(triggerId)
    async(function(triggerId)
        local result = alertSelect(triggerId, {"전사", "마법사", "궁수"}):await()
        if result and result ~= "null" then
            setChatVar(triggerId, "cv_class", result)
            reloadDisplay(triggerId)
        end
    end)
end
```

### 확인 대화상자
```lua
function confirmReset(triggerId)
    async(function(triggerId)
        local ok = alertConfirm(triggerId, "정말 초기화하시겠습니까?"):await()
        if ok then
            setChatVar(triggerId, "cv_step", "0")
            reloadDisplay(triggerId)
        end
    end)
end
```

## 패턴 6: 토글 버튼 (ON/OFF)

설정 패널 등에서 옵션을 켜고 끄는 패턴입니다.

### Lua
```lua
if data == "toggle-option" then
    local current = getChatVar(triggerId, "cv_option") or "0"
    if current == "1" then
        setChatVar(triggerId, "cv_option", "0")
    else
        setChatVar(triggerId, "cv_option", "1")
    end
    reloadDisplay(triggerId)
    return
end
```

### CBS (정규식 OUT)
```html
{{#if {{equal::{{getvar::cv_option}}::1}}}}
<div class="toggle on" risu-btn="toggle-option">ON</div>
{{/if}}
{{#if {{not::{{equal::{{getvar::cv_option}}::1}}}}}}
<div class="toggle off" risu-btn="toggle-option">OFF</div>
{{/if}}
```

**핵심**: ON일 때와 OFF일 때 각각 다른 HTML을 렌더링합니다. 같은 `risu-btn`으로 토글합니다.

## 패턴 7: 리스트에서 선택 (접두사 매칭)

캐릭터 목록, 아이템 목록 등에서 항목을 선택하는 패턴입니다.

### HTML 생성 (Lua)
```lua
function generateItemListHTML(triggerId)
    local items = {"검", "방패", "물약", "활"}
    local html = "<div class=\"my-card-grid\">"
    for i, item in ipairs(items) do
        html = html .. "<div class=\"my-card\" risu-btn=\"item-" .. i .. "\">"
        html = html .. item
        html = html .. "</div>"
    end
    html = html .. "</div>"
    setChatVar(triggerId, "cv_item_list_html", html)
end
```

### 버튼 핸들러 (Lua)
```lua
-- "item-1", "item-2", "item-3" 등을 한 번에 처리
if data:match("^item%-") then
    local idx = tonumber(data:match("item%-(%d+)"))
    local items = {"검", "방패", "물약", "활"}
    if idx and items[idx] then
        setChatVar(triggerId, "cv_selected_item", items[idx])
        setChatVar(triggerId, "cv_step", "2")  -- 다음 단계로
        reloadDisplay(triggerId)
    end
    return
end
```

### CBS (정규식 OUT)
```html
{{#if {{equal::{{getvar::cv_step}}::1}}}}
{{getvar::cv_item_list_html}}
{{/if}}
```

**핵심**: `risu-btn="접두사-인덱스"` → Lua에서 `data:match("^접두사%-")` 패턴 매칭으로 처리.

## 패턴 8: Lua에서 HTML 생성 → chatVar에 저장 → CBS로 표시

복잡한 UI를 CBS만으로 만들기 어려울 때, Lua에서 HTML을 생성하여 chatVar에 저장하고 CBS `{{getvar}}`로 표시하는 패턴입니다.

### 왜 이 패턴을 쓰나?
- CBS의 반복문(`{{#each}}`)은 제한적
- 조건부 로직이 복잡하면 CBS가 너무 길어짐
- Lua에서 데이터 가공 + HTML 빌드가 훨씬 유연

### 흐름
```
Lua: html 문자열 조립 → setChatVar(id, "cv_my_html", html)
  ↓
정규식 OUT: {{getvar::cv_my_html}}
```

### 예시
```lua
function updateStatusHTML(triggerId)
    local hp = getChatVar(triggerId, "cv_hp") or "100"
    local mp = getChatVar(triggerId, "cv_mp") or "50"
    local html = "<div class=\"status\">"
    html = html .. "<div>HP: " .. hp .. "</div>"
    html = html .. "<div>MP: " .. mp .. "</div>"
    html = html .. "</div>"
    setChatVar(triggerId, "cv_status_html", html)
end
```

```html
<!-- 정규식 OUT -->
{{getvar::cv_status_html}}
```

## 패턴 9: 사이드패널 (항상 표시되는 UI)

채팅 옆에 항상 떠 있는 패널을 만드는 패턴입니다. `editdisplay` 정규식으로 모든 메시지에 패널 HTML을 추가합니다.

### 정규식 설정
- **IN:** `\{SIDE_PANEL\}` (또는 특정 태그)
- **type:** `editdisplay`
- **OUT:** 패널 HTML (CBS 조건부 포함)

### 퍼스트 메시지에 태그 삽입
```html
{SIDE_PANEL}
나머지 첫 메시지 내용...
```

이 태그가 있는 메시지가 화면에 표시될 때마다 정규식이 HTML로 변환합니다.

### CSS (고정 위치)
```css
.side-panel {
    position: fixed;
    right: 0;
    top: 50%;
    transform: translateY(-50%);
    width: 280px;
    z-index: 1000;
}
```

## 패턴 10: 접두사 기반 버튼 핸들러 구조

버튼이 많아지면 if-else가 수백 줄이 됩니다. 접두사로 그룹화하면 관리가 쉬워집니다.

### 구조
```lua
function onButtonClick(triggerId, data)

    -- === 시스템 버튼 ===
    if data == "start-btn" then ... return end
    if data == "reset-btn" then ... return end

    -- === 토글 계열 (toggle-*) ===
    if data:match("^toggle%-") then
        local key = data:match("toggle%-(.+)")
        -- cv_key 토글 로직
        return
    end

    -- === 아이템 계열 (item-*) ===
    if data:match("^item%-") then
        local idx = tonumber(data:match("item%-(%d+)"))
        -- 아이템 선택 로직
        return
    end

    -- === 캐릭터 계열 (char-*) ===
    if data:match("^char%-") then
        local idx = tonumber(data:match("char%-(%d+)"))
        -- 캐릭터 선택 로직
        return
    end

end
```

**핵심**: 각 기능 그룹마다 접두사를 정하고, `data:match("^접두사%-")` 패턴으로 분기합니다.

## 정리: 어디에 뭘 쓰나

| 하고 싶은 것 | 쓰는 도구 |
|-------------|-----------|
| 버튼 클릭 처리 | Lua `onButtonClick` + `risu-btn` |
| 텍스트 입력 받기 | Lua `alertInput` + `risu-trigger` + `async` |
| 화면 전환 | chatVar `cv_step` + CBS `{{#if}}` in 정규식 |
| 상태 표시 (HP, 이름 등) | chatVar + CBS `{{getvar}}` in 정규식 |
| 복잡한 UI 생성 | Lua에서 HTML 조립 → chatVar 저장 → CBS `{{getvar}}` |
| AI 응답에서 데이터 추출 | Lua `onOutput` + `string.gmatch` |
| 태그 숨기기 | 정규식 `editdisplay` (빈 OUT) |
| 토큰 절약 | 정규식 `editrequest` + `chat_index` 조건 |
| 항상 표시되는 패널 | 정규식 `editdisplay` + CSS `position:fixed` |
| 설정 로드 | 로어북 `@@dont_activate` + Lua `upsertLocalLoreBook` |
| AI에게 규칙 전달 | 글로벌노트 + `alwaysActive` 로어북 |
