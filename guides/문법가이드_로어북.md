# 문법가이드 로어북

키워드 기반의 동적 컨텍스트 및 세계관 설정 관리

---

## 로어북이란?

로어북은 AI에게 전달될 **설정(Lore)들의 모음**입니다. 캐릭터 설명과 달리, **특정 키워드가 대화에 등장할 때만** 프롬프트에 삽입되어 토큰을 효율적으로 관리할 수 있습니다.

---

## 각 로어북 항목 필드

### 이름

로어북 항목의 제목 및 설명입니다. (관리 및 식별용)

예: "세계관 설정", "주인공 V"

### 활성화 키

주요 매칭 키워드입니다. **쉼표(,)로 구분**합니다.

예: `세계, 세계관, world, setting`

- **정규식 사용:** true일 경우 활성화 키를 정규식으로 해석합니다.

### 배치 순서

삽입 순서입니다.

- 숫자가 **높을수록** 프롬프트 **뒤쪽**에 배치됩니다.
- 토큰 한계로 로어북이 잘릴 경우 **앞 순서(낮은 숫자)부터 제외**됩니다.

### 프롬프트

실제 AI에 전달될 텍스트입니다. **CBS 문법({{}}) 사용이 가능**합니다.

### 조건부 옵션

| 옵션                  | 설명                                                               |
| --------------------- | ------------------------------------------------------------------ |
| 언제나 활성화         | true일 경우 키워드 매칭 없이 **항상** 프롬프트에 삽입              |
| 멀티플 키 & 두번째 키 | true일 때 사용, 활성화 키와 두번째 키가 **모두** 매칭되어야 활성화 |

## 로어북 설정

### 글로벌 설정

- `true`: 설정 → 고급설정에 정의된 설정을 따릅니다. (재귀 검색이 자동으로 활성화됨)
- `false`: 각 캐릭터 별로 설정되는 값을 따릅니다.

### 재귀 검색

켜지면 로어북 프롬프트 안의 내용도 키 매칭에 활용됩니다. (로어북이 또 다른 로어북을 불러올 수 있음)

### 단어 단위 매칭

정확히 단어여야지 매칭됩니다.

- 예) 키: "서진" → "서진" (O), "이서진" (X)

### 로어북 검색 깊이

매칭에 사용하는 채팅의 개수입니다. (유저의 메시지를 포함)

> 주의사항: 오류로 인해 초반 매칭에 안 잡힐 수 있습니다. 메시지 수가 늘어나면 사라집니다.

### 로어북 최대토큰

로어북 프롬프트의 상한가입니다.

- 로어북 프롬프트의 토큰이 최대 토큰을 넘어서면 **배치 순서가 낮은 순서대로** 프롬프트에서 제외됩니다.

### 로어북+

RAG 기능을 위한 더미데이터입니다. (현재 기능 없음)

## 폴더(Folder) 사용법

| 유형      | 설명                                   |
| --------- | -------------------------------------- |
| 폴더      | 다른 로어북 항목을 그룹화하는 컨테이너 |
| 일반 항목 | 실제 내용을 담고 있는 로어북           |

- 폴더 추가 아이콘으로 폴더를 추가할 수 있습니다.
- 드래그를 통해 로어북을 이동시켜 사용할 수 있습니다.

### 폴더 내부 구조 (MCP/프로그래밍 방식)

폴더 식별자는 **`key` 필드**에 `"folder:<UUID v4>"` 형식으로 저장됩니다 (`id` 필드가 아님).

```json
// 폴더 항목
{
  "key": "folder:35c826ba-36aa-46ac-abb3-c2d93714be6b",
  "comment": "🌍 세계관",
  "content": "",
  "mode": "folder"
}

// 폴더에 속한 일반 항목
{
  "key": "키워드1, 키워드2",
  "comment": "항목 이름",
  "content": "내용...",
  "mode": "normal",
  "folder": "folder:35c826ba-36aa-46ac-abb3-c2d93714be6b"
}
```

- 일반 항목의 `folder` 값이 폴더 항목의 `key` 값과 일치하면 해당 폴더 안에 표시됩니다.
- `folder` 필드가 빈 문자열(`""`)이면 최상위(루트)에 표시됩니다.

## CBS (Curly Braced Syntaxes) 사용

로어북 프롬프트에서 CBS 템플릿 문법 `{{}}`를 사용할 수 있습니다.

### CBS 사용 예시

```text
# 캐릭터 상태

- 이름: {{getvar::char_name}}
- 레벨: {{getvar::char_level}}
- HP: {{calc::{{getvar::base_hp}}+{{getvar::bonus_hp}}}}
- 현재 기분: {{random::좋음::보통::나쁨}}

{{#when {{? {{getvar::char_level}}>10}}}}
## 숙련된 전사
당신은 수많은 전투를 겪은 베테랑입니다.
{{/when}}
```

→ CBS 상세: [CBS_QUICK_REF.md](CBS_QUICK_REF.md) 참고

## 로어북 항목 전체 필드

```json
{
  "key": "키워드1, 키워드2",
  "secondkey": "두번째 키워드",
  "comment": "관리용 이름",
  "content": "AI에 전달될 텍스트 (CBS 사용 가능)",
  "mode": "normal | constant | multiple | child | folder",
  "insertorder": 100,
  "alwaysActive": false,
  "selective": false,
  "useRegex": false,
  "folder": "folder:<부모 폴더의 key 값>",
  "id": "uuid",
  "activationPercent": 100,
  "bookVersion": 1,
  "extentions": { "risu_case_sensitive": false }
}
```

> **폴더 항목 구조:** `{ "key": "folder:<UUID v4>", "comment": "폴더이름", "content": "", "mode": "folder" }`
> 폴더 식별자는 **`key` 필드**에 `"folder:xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"` 형식으로 저장됨 (`id` 필드 아님)

**mode 필드 상세:**
| 값 | 설명 |
|------|------|
| `normal` | 기본 모드 — 키워드 매칭 시 활성화 |
| `constant` | 항상 활성화 (`alwaysActive: true`와 유사) |
| `multiple` | 모든 키워드가 매칭되어야 활성화 |
| `child` | 다른 항목의 자식 |
| `folder` | 폴더 컨테이너 (내용 없음) |

## 데코레이터(Decoration) 문법

로어북 항목의 동작을 세밀하게 제어할 수 있는 고급 옵션입니다. **프롬프트 항목 맨 위**에 `@@`로 시작하는 한 줄을 추가하면 해당 효과가 적용됩니다.

| 데코레이터                   | 설명                                                              |
| ---------------------------- | ----------------------------------------------------------------- |
| `@@depth N`                  | 해당 로어북을 N번째 깊이에 삽입 (중요도 조절)                     |
| `@@reverse_depth N`          | 뒤에서 N번째 깊이에 삽입 (덜 중요한 항목)                         |
| `@@activate_only_after N`    | N번째 메시지 이후에만 활성화                                      |
| `@@activate_only_every N`    | N번째마다 주기적으로 활성화                                       |
| `@@role A`                   | 해당 로어북을 특정 역할(A)로 삽입 (`user`, `assistant`, `system`) |
| `@@scan_depth N`             | 지정한 깊이만큼 키워드 검색                                       |
| `@@is_greeting N`            | N번째 인사말로 간주                                               |
| `@@position A`               | 프롬프트 내 위치 지정 (`personality`, `scenario`, `pt_` 등)       |
| `@@ignore_on_max_context`    | 컨텍스트가 가득 찼을 때 무시                                      |
| `@@additional_keys A,B,C...` | 추가 키워드가 있을 때만 활성화                                    |
| `@@exclude_keys A,B,C...`    | 제외 키워드가 있으면 비활성화                                     |
| `@@match_full_word`          | 부분문자열 대신 단어 단위로 키워드를 매칭                         |
| `@@probability N`            | N% 확률로 활성화                                                  |
| `@@activate`                 | 무조건 활성화                                                     |
| `@@dont_activate`            | 무조건 비활성화                                                   |

> **RisuToki 프리뷰 참고:** F5 프리뷰는 `@@depth`, `@@position`, `@@role`, `@@scan_depth`, `@@probability`, `@@activate`, `@@dont_activate`, `@@match_full_word`, `@@additional_keys`, `@@exclude_keys`의 핵심 동작을 반영합니다. `@@probability`는 재현 가능한 결정적 roll로 시뮬레이션되며, 디버그 패널에서 매칭 키·제외 키·데코레이터 태그·경고를 확인할 수 있습니다.

### 사용 예시

```text
@@role user
@@depth 0
@@activate_only_after 5

이 캐릭터는 플레이어가 직접 조작하는 주인공입니다...
```
