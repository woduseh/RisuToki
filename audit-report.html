A. 개요 & 기술 스택

AI 캐릭터 파일(.charx) 편집기로, 로어북·정규식·Lua 스크립트·CSS/HTML 편집, 실시간 프리뷰, MCP 프로토콜 기반 외부 AI 어시스턴트 통합을 지원하는 데스크톱 도구 애플리케이션. 별도 팝아웃 윈도우로 채팅 프리뷰와 디버그 패널을 제공하며, 로컬 HTTP API 서버를 통한 MCP 도구 연동이 핵심 차별점이다.

C. 주요 발견사항

1. 코드 품질 & 유지보수성 — 7.5/10

🟢 80+ 모듈의 명확한 관심사 분리
src/lib/에 80개 이상의 특화 모듈이 단일 책임으로 분리됨. file-actions.ts, sidebar-builder.ts, layout-manager.ts, preview-engine.ts 등 각각 명확한 영역 담당. Pinia 스토어(app-store.ts)가 21개 반응형 상태를 중앙 관리하며, 액션 레지스트리(action-registry.ts)가 Vue↔컨트롤러 브릿지를 제공.
🟠 controller.ts 모놀리스 (2,364줄) — MAJOR [TRADE-OFF: 점진적 마이그레이션 전략]
src/app/controller.ts 81KB, 프로젝트 최대 파일. 사이드바 구축 로직(~600줄), 에디터 생명주기, 탭 관리, 키보드 단축키가 혼재. registerActions()로 50+ 액션을 Vue에 위임하는 브릿지 패턴이 도입되었으나, DOM 이벤트 리스너 설정(getElementById 97회)이 여전히 명령형. Pinia 전환이 부분적이어서 상태 동기화 보일러플레이트(syncStoreState)가 필요한 구조.
🟠 any 타입 132회 사용, ESLint 규칙 비활성화 — MAJOR [TRADE-OFF: 레거시 마이그레이션 비용]
eslint.config.mjs에서 @typescript-eslint/no-explicit-any: 'off' 설정. strict: true인 tsconfig에도 불구하고 controller.ts에 44회 집중. 근본 원인: let fileData: any = null(L128) — 이 선언이 40+ 파생 사용처로 전파됨. CharxData 인터페이스가 app-store.ts에 이미 정의되어 있어 타입 적용이 가능한 상태.

2. 보안 & 신뢰성 — 6.0/10

🔴 경로 탐색 취약점 — 페르소나 파일 핸들러 — CRITICAL [OVERSIGHT]
main.ts L920-935: read-persona/write-persona IPC 핸들러가 name 파라미터를 검증 없이 path.join()에 전달. ../../.env 같은 입력으로 임의 파일 읽기/쓰기 가능. add-asset-buffer의 targetFolder도 동일 문제.
🟠 innerHTML XSS — 프리뷰 콘텐츠 미이스케이프 — MAJOR [OVERSIGHT]
src/lib/preview-format.ts L252: buildPreviewMessageHtml()에서 name은 escapePreviewHtml()로 이스케이프하면서 content는 원본 삽입. AI 응답 내용이 HTML로 렌더링되는 경로. 66개 innerHTML 사용 중 대부분은 정적 한국어 문자열(저위험)이나, 이 지점은 동적 데이터.
🟢 MCP API 서버 보안 강화 완료

🟡 DOMPurify CVE 2건 (moderate) — MINOR [OVERSIGHT]
npm audit: monaco-editor가 의존하는 dompurify 3.1.3-3.3.1에 XSS 취약점(GHSA-v2wj-7wpq-c8vv). 직접 사용은 아니나 의존성 체인에 포함.
