A. 개요 및 기술 스택

데스크톱 캐릭터 카드 편집기 애플리케이션으로, 특정 AI 챗봇 플랫폼의 .charx 파일을 시각적으로 편집하며 통합 터미널과 MCP API 서버를 제공한다. 메인 프로세스에서 HTTP API를 노출해 외부 AI 어시스턴트가 캐릭터 데이터를 읽고 쓸 수 있도록 하는 구조가 핵심이다. 최근 대규모 모듈화 및 TypeScript 마이그레이션을 거쳐 레거시 JS 코드 ~9,000줄이 제거되고, 모놀리식 파일이 64개 모듈로 분리되었다.

근거: controller.ts 47건, mcp-api-server.ts 9건, asset-manager.ts 9건, data-serializer.ts 7건

2. 보안 & 신뢰성 — 7.0 / 10

🔵 강점 Electron 보안 모범 사례 준수

contextIsolation: true, nodeIntegration: false로 렌더러-메인 프로세스 격리. contextBridge.exposeInMainWorld로 명시적 IPC API만 노출. MCP API에 crypto.randomBytes(32) Bearer 토큰 인증과 모든 mutation에 사용자 확인 다이얼로그 적용.

근거: main.ts L436-439, preload.ts contextBridge, mcp-api-server.ts L193·L206·L273-276
🟠 주요 Object.assign 필드 필터링 부재 (CWE-915) OVERSIGHT

로어북/정규식 수정 시 Object.assign(entry, body)로 요청 본문 전체를 병합. 공격자가 __proto__, constructor 또는 임의의 내부 속성을 주입할 수 있는 prototype pollution 벡터. 허용 필드 화이트리스트로 필터링 필요.

근거: mcp-api-server.ts L385, L458 (regex도 동일 패턴)
🟡 경미 readBody 크기 제한 없음 OVERSIGHT

HTTP 요청 본문을 무한히 버퍼링. 로컬 네트워크 전용이지만 악의적 클라이언트(또는 오작동하는 MCP 도구)가 대용량 본문으로 메모리 소진 가능.

근거: mcp-api-server.ts L63-69, 청크 크기 확인·최대 길이 체크 없음
🟡 경미 프로덕션 의존성 CVE 2건 OVERSIGHT

monaco-editor ≥0.54의 내부 의존성 dompurify 3.1.3-3.3.1에 XSS 취약점(GHSA-v2wj-7wpq-c8vv, moderate). Monaco를 코드 에디터로만 사용하므로 실제 위험은 낮으나 패치 필요.

근거: npm audit --omit=dev 결과, package.json L34 monaco-editor ^0.55.1

🟡 경미 리팩토링 회피 (Refactoring avoidance)

모듈 추출은 완료되었으나 controller.ts의 기존 명령형 DOM 조작 패턴을 그대로 유지한 채 분리. Vue 반응형 시스템으로의 전환 없이 레거시 패턴을 모듈 단위로 감싸기만 한 형태. 장기적으로 두 번째 리팩토링이 불가피.

빈도: 전체 lib 모듈 64개 중 대다수가 DOM 직접 조작. 위험: 유지보수 비용 증가, 새 개발자 학습 곡선 상승.
🔵 제안 과잉 명세 (Over-specification) — 경미

일부 deps 인터페이스가 단일 구현체만 존재함에도 세밀한 getter 함수 패턴을 적용 (예: getDirname: () => __dirname). 테스트 편의를 위한 설계이나 실제 모킹 대상이 아닌 항목까지 포함.

빈도: 11개 메인 프로세스 모듈의 deps 인터페이스. 실제 위험: 낮음 (코드 양 미미하게 증가).

핵심 기능은 작동하고 CI 파이프라인도 갖춰져 있으므로 현 상태에서 배포 자체는 가능하나, any 제거·테스트 보강·API 입력 검증 강화를 거쳐야 유지보수 가능한 프로덕션 수준에 도달한다.
